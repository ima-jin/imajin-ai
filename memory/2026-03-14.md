## #345 — Inline File Editor for .imajin Files

### What was built

**New API endpoint:**
- `apps/media/app/api/assets/[id]/content/route.ts` — `PUT` saves text file content back to disk, updates `hash` + `size` in DB, owner-only

**New component:**
- `apps/media/src/components/FileEditor.tsx` — `<FileEditor asset isOwner>` client component
  - Lazy-loads `react-simple-code-editor` + `prismjs` + `react-markdown`
  - Syntax highlighting for JSON and Markdown via prismjs
  - Edit/Preview toggle (Edit hidden for non-owners)
  - Ctrl/Cmd+S keyboard shortcut to save
  - Markdown preview via `react-markdown` with dark prose styles
  - JSON preview with pretty-print formatting
  - Read-only badge shown for non-owners
  - Also exports `isTextAsset(asset)` helper to detect text files

**Modified components:**
- `apps/media/src/components/AssetDetail.tsx` — accepts `currentDid?: string`, shows `<FileEditor>` instead of download fallback for text files
- `apps/media/src/components/MediaManager.tsx` — passes `session.id` as `currentDid` to `AssetDetail`

**New packages installed in `apps/media`:**
- `react-simple-code-editor`, `prismjs`, `react-markdown`, `@types/prismjs`
