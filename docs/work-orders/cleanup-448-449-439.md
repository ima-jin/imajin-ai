## Step 1: #448 — Move emitAttestation to @imajin/auth

### What to do

1. **Create** `packages/auth/src/emit-attestation.ts`:

```typescript
export async function emitAttestation(params: {
  issuer_did: string;
  subject_did: string;
  type: string;
  context_id: string;
  context_type: string;
  payload?: Record<string, unknown>;
}): Promise<void> {
  const authServiceUrl = process.env.AUTH_SERVICE_URL;
  const internalApiKey = process.env.AUTH_INTERNAL_API_KEY;
  if (!authServiceUrl || !internalApiKey) {
    console.warn('Attestation skipped: AUTH_SERVICE_URL or AUTH_INTERNAL_API_KEY not set');
    return;
  }
  try {
    const res = await fetch(`${authServiceUrl}/api/attestations/internal`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${internalApiKey}`,
      },
      body: JSON.stringify(params),
    });
    if (!res.ok) {
      const text = await res.text().catch(() => '');
      console.error(`Attestation (${params.type}) failed: ${res.status} ${text}`);
    }
  } catch (err) {
    console.error(`Attestation (${params.type}) error:`, err);
  }
}
```

2. **Export** from `packages/auth/src/index.ts`:
```typescript
export { emitAttestation } from "./emit-attestation";
```

3. **Delete** these files:

4. **Update imports** in these files (change to `import { emitAttestation } from '@imajin/auth'`):

   - `apps/connections/app/api/invites/[code]/accept/route.ts`
   - `apps/connections/app/api/invites/route.ts`
   - `apps/connections/app/api/groups/[id]/members/route.ts` (if it imports attestations)
   - `apps/connections/app/api/pods/[id]/members/route.ts` (if it imports attestations)

5. **Refactor** `apps/pay/app/api/settle/route.ts`:
   - Import `emitAttestation` from `@imajin/auth`
   - Replace the inline `emitAttestations` function to call the shared `emitAttestation` in a loop

6. **Verify:** Build events, connections, and pay:

## Step 3: #439 — Migrate media URL strings to asset ID references

### Context

9 services store media references as raw URL strings. These should be asset IDs (`asset_xxx`) that resolve at render time, enabling reference tracking, revocation, and CID addressing.

### What to do

**For each service, the pattern is:**

1. Add `assetId` column (nullable text) alongside existing URL column
2. Create a shared resolver: `resolveAssetUrl(assetId: string): string` in `@imajin/fair` or a new `@imajin/media` package
3. Update write paths: when uploading, store the returned `asset_xxx` ID instead of (or alongside) the URL
4. Update read paths: resolve asset ID → URL at render time
5. Backfill script: parse existing URLs to extract asset IDs where possible

### Services to migrate

| Service | Table/Field | Current Type | New Field |
|---------|------------|-------------|-----------|
| market | listings.images | jsonb URLs | listings.imageAssetIds (jsonb) |
| events | events.imageUrl | text URL | events.imageAssetId (text) |
| learn | courses.imageUrl | text URL | courses.imageAssetId (text) |
| auth | identities.avatarUrl | text URL | identities.avatarAssetId (text) |
| profile | profiles.avatar | text URL/emoji | profiles.avatarAssetId (text, nullable — emoji stays in avatar) |
| coffee | pages.avatar | text URL/emoji | pages.avatarAssetId (text, nullable) |
| links | pages.avatar | text URL/emoji | pages.avatarAssetId (text, nullable) |
| connections | (check schema) | varies | varies |
| chat | messages_v2.mediaPath | text path | messages_v2.mediaAssetId (text) |

### Reference registration

Add to media service:

```typescript
// POST /api/assets/{id}/references
// { service: 'events', entityType: 'event', entityId: 'evt_xxx' }
```

And a reference table:
```sql
CREATE TABLE media.asset_references (
  id TEXT PRIMARY KEY,
  asset_id TEXT NOT NULL REFERENCES media.assets(id),
  service TEXT NOT NULL,
  entity_type TEXT NOT NULL,
  entity_id TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(asset_id, service, entity_type, entity_id)
);
```

### Shared URL resolver

Create `packages/media/src/resolve.ts` (or add to existing package):

```typescript
export function resolveAssetUrl(assetId: string, mediaBaseUrl: string): string {
  return `${mediaBaseUrl}/api/assets/${assetId}/content`;
}
```

Components use this instead of storing raw URLs.

### Migration order

Do services one at a time, smallest first:
1. coffee (1 field, low traffic)
2. links (1 field)
3. dykil (check if any)
4. learn (1 field)
5. events (1 field, higher traffic)
6. profile (avatar — shared with auth)
7. auth (avatar)
8. market (jsonb array — more complex)
9. chat (mediaPath → assetId)

### Backfill

Write `scripts/backfill-asset-refs.ts`:
- For each service, query rows with URL but no assetId
- Parse URL to extract asset ID (e.g. `/api/assets/asset_xxx/content` → `asset_xxx`)
- Update the row with the extracted ID
- Register the reference with media

### Build & verify

Build each service after migration. Run backfill in --dry-run first.

**Commit per service** (don't do all 9 in one commit):

- `refactor(links): migrate avatar URL to asset ID (#439)`
- etc.

## Future: CID as asset identity

The asset ID column (`asset_id TEXT`) is deliberately opaque. Currently stores `asset_xxx` strings. A future media-internal change will switch to content-addressed CIDs (derived from file hash via dag-cbor). This is scoped to media service only — the reference columns in other services won't need to change since they just store whatever ID media returns. See #400 and RFC-18 (Discussion #438).
